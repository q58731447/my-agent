當使用者提供學習紀錄時，標準流程如下：

1. 存入 Heptabase（Note Card + Journal）
2. 依格式草擬 Threads 貼文，**展示給使用者確認**
3. 詢問：「是否可以 PO 到 Threads？」
4. **使用者同意後才發文**，絕不自動 PO

**Threads 貼文格式：**
- 第一區塊：完整保留原始標題行與 🎯 今天目標區塊（emoji、✅❌符號、條目原文不動）
- 第二區塊：以**敘事段落風格**呈現今日成果與心得，不用條列式
- 結尾：一句話帶出下一個挑戰或方向
- **不加 hashtag**

**範例結構：**
```
學習紀錄 | YYYY.MM.DD Day XX

🎯 今天目標：
• 任務一 ✅/❌
• 任務二 ✅/❌

[一句話點出今日亮點]

[敘事段落：說明完成了什麼、意義是什麼]

[結尾：下一步挑戰或感受]
```

**Why:** 使用者確認過此格式（Day 33），標題與目標原文保留，內文用段落敘事而非條列，讀起來更像個人日記風格。

**How to apply:** 每次收到學習紀錄，存完 Heptabase 後，依此格式草擬貼文並問「可以發嗎？」，等確認後再執行發文。
